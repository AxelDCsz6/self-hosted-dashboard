# Security Policy

## Reporting Security Issues

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please contact us directly at axeldc585@gmail.com 

## Security Measures

- CSRF protection on all forms
- Secure session configuration with HTTP-only cookies
- SQL injection protection via prepared statements
- XSS prevention through output escaping
- Role-based access control

## Best Practices

- Change default credentials after installation
- Use HTTPS in production
- Regular security updates
- Database file stored outside web root
